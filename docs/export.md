# Shot Caller VR — Call Sheet & Shot List Export

**Last updated:** March 15, 2026

---

## Overview

The export stage converts the 3D blocked scene into a production-ready document that a filmmaker can hand to cast and crew on the day of the shoot. It combines the spatial data from the editor (where cameras, actors, and equipment are placed) with AI-generated production logic (call times, cost estimates, shot descriptions).

---

## Document Structure

### 1. Production Header

| Field | Source |
|-------|--------|
| Project Title | Scene name (set during intake) |
| Shoot Date | User input |
| Location | Address from Google Maps / intake |
| Director | User input |
| 1st AD | User input |
| General Call | Derived from earliest cast/crew call time |

---

### 2. Shot List

One row per camera element, ordered by shot number.

| Col | Source |
|-----|--------|
| Shot # | `camera.shotNumber` |
| Setup | `camera.setupGroup` (A/B/C/D) |
| Shot Type | `camera.shotType` (Wide / Medium / CU / OTS / Insert / POV) |
| Description | AI-generated from camera position, target, and shot type |
| Lens (FOV) | `camera.vFovDeg` → converted to approximate mm equivalent |
| Est. Duration | AI-estimated based on shot complexity |
| Notes | `camera.shotLabel` |

**Setup Group ordering:** Shots are grouped by setup (A, B, C, D) to minimize equipment resets. Within each group, shots are ordered by shot number.

**Example:**

| Shot | Setup | Type | Description | Lens | Est. |
|------|-------|------|-------------|------|------|
| 1 | A | Wide | Establishing — full alley from south end | 24mm | 3 min |
| 2 | A | Medium | Two-shot — Mark and Elena at dumpster | 50mm | 5 min |
| 3 | A | CU | Insert — key slides into lock | 85mm | 8 min |
| 4 | B | OTS | Over Elena's shoulder toward doorway | 35mm | 4 min |

---

### 3. Cast Call Sheet

One section per cast mark element in the scene.

| Field | Source |
|-------|--------|
| Character | `castMark.name` |
| Actor | `castMark.properties.actor` |
| Call Time | AI-derived (based on which setup they first appear in, minus prep time) |
| On Set | AI-derived (first shot they appear in) |
| Wrap | AI-derived (last shot they appear in + buffer) |
| Notes | Costume / makeup notes if provided |

**Call time logic:**
- Base call = general call time
- Cast appearing in Setup A only: on set by general call
- Cast appearing in Setup B+: staggered 30–60 min later
- Hair/makeup buffer: +45 min before on-set time (configurable)

---

### 4. Crew List

One row per crew element, grouped by department.

| Field | Source |
|-------|--------|
| Name | `crew.name` |
| Role | `crew.properties.role` |
| Department | AI-grouped (Camera, Grip, Electric, Art, Sound, etc.) |
| Call Time | General call (crew), or setup-specific for department heads |
| Notes | |

---

### 5. Equipment List

One row per equipment element, plus camera and light elements.

| Field | Source |
|-------|--------|
| Item | Element name |
| Type | Camera / Light / Grip / Props |
| Qty | `equipment.properties.quantity` |
| Setup | Setup group it's used in |
| Day Rate | AI-estimated based on item name/type |
| Subtotal | Qty × Day Rate |

**Cost estimation:**
- Claude looks up approximate industry day rates based on equipment name
- Example: "ARRI Alexa 35" → $1,200/day, "Aputure 600d" → $150/day, "15' HMI" → $400/day
- Rates are estimates only — clearly marked as such in the document

---

### 6. Props List

One row per props element.

| Field | Source |
|-------|--------|
| Item | `props.name` |
| Description | `props.properties.description` |
| Qty | `props.properties.quantity` |
| Dimensions | W × H × D from props element |
| Notes | Source / prep notes |

---

### 7. Budget Summary

| Category | Estimate |
|----------|----------|
| Camera Package | $X,XXX |
| Lighting & Grip | $X,XXX |
| Props | $XXX |
| Location Fee | User input |
| Catering (est.) | Based on crew count × $X/head |
| **Total (est.)** | **$X,XXX** |

All figures are clearly marked as AI-generated estimates. The export includes a disclaimer that rates vary by market and vendor.

---

### 8. Schedule / Day Overview

A timeline view of the shoot day, generated by Claude:

```
06:00  Crew call — Grip & Electric, Art Department
07:00  General crew call
07:30  Camera tests, lighting setup (Setup A)
08:30  Cast call — Mark, Elena
09:00  SHOOT — Setup A (Shots 1, 2, 3) — est. 2.5 hrs
11:30  Setup B — reset lighting, reposition camera
12:00  LUNCH — 30 min
12:30  SHOOT — Setup B (Shots 4, 5) — est. 1.5 hrs
14:00  Wrap Setup B / Load out
15:00  WRAP
```

## AI Generation Prompt

The export endpoint sends a structured JSON payload to Claude:

```json
{
  "scene": {
    "name": "Alley - Night",
    "location": "123 Main St, Los Angeles, CA",
    "cameras": [...],
    "castMarks": [...],
    "crew": [...],
    "equipment": [...],
    "props": [...]
  },
  "userInputs": {
    "shootDate": "2026-04-15",
    "generalCall": "07:00",
    "director": "Mark Harmon",
    "lunchDuration": 30
  }
}
```

Claude returns structured JSON with:
- Shot descriptions
- Call times per cast/crew member
- Equipment day rate estimates
- Schedule blocks
- Any continuity or logistical flags (e.g. "Setup C requires a lighting reset that may take 45+ min")

## Export Formats

| Format | How |
|--------|-----|
| **On-screen HTML** | Rendered inline in the editor as a fullscreen overlay |
| **PDF** | `window.print()` with a dedicated print stylesheet (portrait, letter/A4) |
| **JSON** | Raw structured output, downloadable for import into Movie Magic, StudioBinder, etc. |
| **Markdown** | Plain text version for sharing via Slack, email, or GitHub |

---

## Implementation Notes

- The call sheet is generated client-side using the Anthropic SDK (same pattern as the intake agent)
- No server round-trip required unless PDF generation needs Puppeteer
- Scene JSON must pass through `SceneState.save()` before export so all element data is fresh
- Equipment cost estimates should always include a "rates as of [date], verify with vendor" disclaimer
- The document should be printable on a single page per section (shot list, cast, crew, equipment) using CSS `@page` breaks
